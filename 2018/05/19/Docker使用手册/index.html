Docker使用手册

Docker简介

Docker 是一个开源的应用容器引擎，基于 Go 语言 并遵从Apache2.0协议开源。

Docker 可以让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。

容器是完全使用沙箱机制，相互之间不会有任何接口（类似 iPhone 的 app）,更重要的是容器性能开销极低。

Docker常用指令介绍

原理：先用docker ps -a -q 输出所有容器的container id(-f 表示过滤参数或者输出格式)，然后作为docker rm 的参数进行批量删除

  1. 输出所有容器的name：

    docker ps --format='{{.Names}}'
test-env
test-args
test-run
输出所有容器名包含test的容器，并打印容器名
docker ps -f name=test --format='{{.Names}}'
test-env
test-args
test-run
查看退出状态的容器，并打印容器名
docker ps -f status=exited --format="{{.Names}}"
thirsty_brahmagupta
clever_mestorf
hopeful_morse
stoic_morse
elated_williams
tender_jepsen
reverent_mirzakhani

  2. 删除所有容器：

    docker rm -f -v $(docker ps -a -q)
删除/启动所有退出的容器：
docker rm/start $(docker ps -qf status=exited)
删除所有镜像：
docker rmi $(docker images -q)

  3. 其他：

    查看悬挂镜像:
docker  images -qf dangling=true

只查看镜像或者容器指定的信息(在docker1.10之后才支持的)

只列出镜像的id以及仓库名称：
docker images --format "{{.ID}}: {{.Repository}}"
67591570dd29: centos
0a18f1c0ead2: rancher/server

只列出容器的相关id,image,status和name
docker ps --format "{{.ID}}: {{.Image}} : {{.Status}} : {{.Names}}"
66b60b72f00e: centos : Up 7 days : pensive_poincare
或者自己重新定义列,就和原生差不多:
docker ps --format "table {{.ID}}\t{{.Image}}\t{{.Status}}\t{{.Names}}"
CONTAINER ID        IMAGE                                         STATUS              NAMES
66b60b72f00e        centos                                        Up 7 days           pensive_poincar

如何使用阿里云Docker（新版旧版升级）

如下代码，都以docker为例

新版本发布

运行项目

运行如下指令：

docker run -d -it --name docker.0.0.1 -e 'CONFIG_HOST=127.0.0.1' -e 'CONFIG_PORT=80' -e 'CONFIG_LABEL=label' -e 'CONFIG_NAME=docker' -e 'CONFIG_PROFILE=prod'  -p 80:80 docker rmi registry.cn-beijing.aliyuncs.com/docker:0.0.1

版本升级

停止旧的Docker项目

停止旧的Docker版本：

docker stop docker.0.0.1

运行新的Docker项目

运行如下指令：

docker run -d -it -e CONFIG_PROFILE=prod --name docker.0.0.2 -p 9091:80 registry.cn-beijing.aliyuncs.com/docker:0.0.2

删除旧的Docker项目

查看项目是否正常显示。如正常删除旧的Docker版本镜像：

docker rm docker:0.0.1
docker rmi registry.cn-beijing.aliyuncs.com/docker:0.0.1

如何设置阿里云Docker加速

修改/etc/docker/daemon.json 文件 (centos7系统），如无新增。

"registry-mirrors": ["https://123456.mirror.aliyuncs.com"]

https://123456.mirror.aliyuncs.com这个地址需要自己去阿里云注册个账号后，进入自己的控制台，获取国内加速地址。

ECS通过内网推送镜像

每个阿里云账户的username不一样，每个区的registry地址不一样，请根据情况填写相关地址

如何登陆阿里云docker registry:

$ sudo docker login --username=username registry.cn-hangzhou.aliyuncs.com

登录registry的用户名是您的阿里云账号全名，密码是您开通服务时设置的密码。

你可以在镜像管理首页点击右上角按钮修改docker login密码。
